5. Создание политик и задач для антивирусных клиентов
5.1 Создание политики
Для создания политики для Kaspersky Endpoint Security для Linux необходимо выполнить следующие действия:
1.	В главном окне программы перейти в раздел /Активы (Устройства) /Политики и профили политик.
2.	Нажать кнопку «Добавить».
3.	Откроется окно «Выбрать программу».
4.	Выбрать программу, для которой требуется создать политику
5.	Нажать кнопку «Далее».
6.	Выбрать «Автономный режим».
7.	На этапе «Участие в Kaspersky Security Network», поставьте галочку «Я не принимаю условия использования Kaspersky Security Network».
8.	Откроется окно параметров новой политики на вкладке «Общие».
9.	При необходимости изменить следующие параметры политики, заданные по умолчанию: имя, состояние и наследование.
10.	Выбрать вкладку «Параметры программы» или нажать кнопку «Сохранить», чтобы выйти.

5.2 Создание задачи обновления
Для создания задачи Обновления для Kaspersky Endpoint Security для Linux необходимо выполнить следующие действия:
1.	В главном окне Web Console выбрать Активы (Устройства) → Задачи.
2.	Нажать кнопку «Добавить».
3.	На первом шаге мастера выполнить следующие действия:
4.	В раскрывающемся списке «Программа» выбрать Kaspersky Endpoint Security для Linux.
5.	В раскрывающемся списке «Тип задачи» выбрать тип задачи «Обновление».
6.	В поле «Название» задачи ввести название новой задачи.
7.	В блоке «Устройства, которым будет назначена задача» выбрать способ определения области действия задачи. Область действия задачи – это устройства, на которых будет выполняться задача:
	Выбрать вариант «Назначить задачу группе администрирования», если задача должна выполняться на всех устройствах, входящих в определенную группу администрирования.
	Выбрать вариант «Задать адреса устройств вручную или импортировать из списка», если задача должна выполняться на указанных устройствах.
	Выбрать вариант «Назначить задачу выборке устройств», если задача должна выполняться на устройствах, входящих в выборку устройств по предопределенному критерию.
8.	Нажать кнопку «Далее».
Чтобы настроить параметры задачи сразу после создания, на последнем шаге мастера  необходимо установить флажок «Открыть окно свойств задачи после ее создания». Задача создается с параметрами по умолчанию.
9.	Нажать кнопку «Готово». В списке задач отобразится новая задача «Обновление»
10.	Выбрать задачу «Обновление» и откройте окно свойств задачи по ссылке в названии задачи.
11.	В окне свойств задачи выбрать закладку «Параметры приложения», в списке слева. Выбрать раздел «Источники обновлений».
12.	Выбрать источник обновлений, из которого приложение будет получать обновление баз и модулей, в зависимости от используемой схемы обновления.
	При управление приложением с помощью Web Console, список источников обновлений содержит серверы обновлений "Лаборатории Касперского" и Сервер администрирования Kaspersky Security Center. Если приложение управляется с помощью Kaspersky Security Center Cloud Console, список источников обновлений содержит серверы обновлений "Лаборатории Касперского" и точки распространения (подробнее о точках распространения см. в справке Kaspersky Security Center). При необходимости можно добавлять в список другие источники обновлений.
	Сформировать список источников обновления, выбрав вариант «Другие источники в локальной или глобальной сети». В качестве источников обновлений указывать FTP-, HTTP- или HTTPS-серверы. Если обновление не может быть выполнено из одного источника обновлений, приложение Kaspersky Endpoint Security переключается к следующему источнику.
13.	Перейти в раздел «Параметры» и настройте другие параметры обновления.
14.	Выбрать закладку «Расписание» и настройте расписание запуска задачи обновления.
Если в качестве источника обновлений выбран «Kaspersky Security Center», в раскрывающемся списке запуск по расписанию выбрать «При загрузке обновлений в хранилище» или по времени. 
15.	Нажать кнопку «Сохранить», чтобы сохранить внесенные изменения. 

5.3 Создание задачи Поиска вредоносного ПО
Создать задачу «Поиск вредоносного ПО», действия аналогичны задаче «Обновление»
5.4 Создание задачи «Добавление ключа»
Добавить ключ в хранилище ключей KSC. В главном окне Web Console выбрать /Операции /Лицензии "Лаборатории Касперского".
Создать задачу «Добавление ключа», действия аналогичны задаче «Обновление»
