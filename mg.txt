Ниже представлены два сценария настройки, которые решают вашу задачу: пропускать письма только с определенного домена (белый список) или перехватывать и блокировать все остальные.

Сценарий 1: Пропускать письма ТОЛЬКО с определенного домена (Белый список)
Этот сценарий подходит, если ваш почтовый шлюз должен принимать и обрабатывать сообщения исключительно от отправителей из доверенного домена (например, example.com), а письма от всех остальных доменов отклонять или удалять.

Шаг 1. Создайте правило для разрешенного домена
В веб-интерфейсе KSMG перейдите в раздел Правила.

Нажмите кнопку Создать.

На вкладке Информация о правиле:

Включите переключатель Статус.

В поле Название правила введите, например, Allow Only Trusted Domain.

В блоке Режим выберите Использовать параметры модулей проверки. Это позволит KSMG сканировать письма на вирусы и спам, но только для выбранных отправителей.

Приоритет оставьте по умолчанию (наивысший).

Перейдите на вкладку Отправители и получатели.

В блоке Отправитель перейдите на вкладку Адреса эл. почты и нажмите Добавить.

Введите маску домена, с которого разрешен прием писем. Например: *@example.com (используйте символ * для обозначения любого пользователя). Или используйте вкладку IP, если знаете IP-адреса почтовых серверов доверенного домена.

В блоке Получатель перейдите на вкладку Адреса эл. почты и нажмите Добавить. Введите * (звездочку), чтобы правило применялось для всех ваших внутренних получателей.

Нажмите Сохранить. Правило создано, но пока отключено.

Шаг 2. Создайте правило для блокировки всех остальных
Теперь нужно создать правило по умолчанию, которое будет обрабатывать всех "левых" отправителей.

Снова нажмите Создать.

Название правила: Block All Others.

В блоке Режим на этот раз выберите Отклонять без проверки (или Удалять без уведомления отправителя, если не хотите отправлять отчеты о недоставке).

На вкладке Отправители и получатели:

Отправитель → Адреса эл. почты: Добавьте * (все возможные отправители).

Получатель → Адреса эл. почты: Добавьте * (все возможные получатели).

Нажмите Сохранить.

Шаг 3. Включите правила
Убедитесь, что переключатель Статус в строке правила Allow Only Trusted Domain находится в положении Вкл. (зеленый).

Убедитесь, что переключатель в строке правила Block All Others также включен.

Результат: Вся почта будет обрабатываться по порядку. Сначала KSMG проверит, пришло ли письмо с *@example.com для любого вашего получателя (*). Если да, оно будет принято и проверено. Если нет, сработает второе правило, и письмо будет отклонено без проверки.

Сценарий 2: "Перехват" писем с копированием (BCC) и анализом
Термин "перехват" может означать сохранение копии всех писем. Для этого используется функция Скрытая копия (BCC).

Шаг 1. Создайте правило для перехвата всего трафика
Перейдите в раздел Правила → Создать.

Название правила: Archive All Email.

Режим: Использовать параметры модулей проверки (или выберите Пропускать без проверки, если копии должны сохраняться до сканирования).

На вкладке Отправители и получатели укажите:

Отправитель: *

Получатель: *

Перейдите на вкладку Скрытая копия (BCC).

Нажмите Добавить.

Введите адрес электронной почты, на который нужно отправлять копии всех писем (например, archive@your-company.com). На этот адрес будет приходить точная копия каждого письма, проходящего через шлюз.

Нажмите Сохранить и включите правило.

Важно: Согласно документации, при отправке скрытой копии KSMG подменяет адрес отправителя в SMTP-сессии на адрес приложения (настраивается в Параметры → Уведомления → Адрес отправителя уведомлений), чтобы исходный отправитель не получил случайный отчет о недоставке, если BCC-адрес окажется невалидным.
